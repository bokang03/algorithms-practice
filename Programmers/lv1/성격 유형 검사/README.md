# [Programmers] 성격 유형 검사하기

### 문제 설명
나만의 카카오 성격 유형 검사지를 만들려고 합니다. 성격 유형 검사는 다음과 같은 4개 지표로 성격 유형을 구분합니다. 성격은 각 지표에서 두 유형 중 하나로 결정됩니다.

| 지표 번호 | 성격 유형 |
| :--- | :--- |
| 1번 지표 | 라이언형(R), 튜브형(T) |
| 2번 지표 | 콘형(C), 프로도형(F) |
| 3번 지표 | 제이지형(J), 무지형(M) |
| 4번 지표 | 어피치형(A), 네오형(N) |

4개의 지표가 있으므로 성격 유형은 총 16(=2 x 2 x 2 x 2)가지가 나올 수 있습니다. 검사지에는 총 `n`개의 질문이 있고, 각 질문에는 7개의 선택지가 있습니다.

* 매우 비동의 / 비동의 / 약간 비동의 / 모르겠음 / 약간 동의 / 동의 / 매우 동의

각 질문은 1가지 지표로 성격 유형 점수를 판단합니다. 선택지에 따른 점수는 다음과 같습니다.

| 선택지 | 성격 유형 점수 |
| :--- | :--- |
| 매우 비동의 | 3점 |
| 비동의 | 2점 |
| 약간 비동의 | 1점 |
| 모르겠음 | 0점 |
| 약간 동의 | 1점 |
| 동의 | 2점 |
| 매우 동의 | 3점 |

검사 결과는 모든 질문의 점수를 더하여 각 지표에서 더 높은 점수를 받은 성격 유형으로 판단합니다. 단, 점수가 같으면 **사전 순으로 빠른** 성격 유형을 선택합니다.

### 제한사항
* 1 ≤ `survey`의 길이 ( = `n`) ≤ 1,000
* `survey`의 원소는 `"RT", "TR", "FC", "CF", "MJ", "JM", "AN", "NA"` 중 하나입니다.
  * `survey[i]`의 첫 번째 캐릭터: 비동의 관련 선택지 선택 시 받는 유형
  * `survey[i]`의 두 번째 캐릭터: 동의 관련 선택지 선택 시 받는 유형
* `choices`의 길이 = `survey`의 길이
  * `choices[i]`는 1~7 사이의 정수 (1: 매우 비동의, 4: 모르겠음, 7: 매우 동의)

### 입출력 예
| survey | choices | result |
| :--- | :--- | :--- |
| ["AN", "CF", "MJ", "RT", "NA"] | [5, 3, 2, 7, 5] | "TCMA" |
| ["TR", "RT", "TR"] | [7, 1, 3] | "RCJA" |

### 입출력 예 설명
**입출력 예 #1**
1번부터 5번까지 질문의 성격 유형 점수를 합산하면 다음과 같습니다.
* 1번 지표: R(0), T(3) → **T**
* 2번 지표: C(1), F(0) → **C**
* 3번 지표: J(0), M(2) → **M**
* 4번 지표: A(1), N(1) → 점수 동일, 사전 순 **A**
결과: "TCMA"

**입출력 예 #2**
모든 지표를 합산한 결과 1번 지표는 R이 높고, 나머지는 모두 0점으로 동일합니다. 0점일 때는 사전 순으로 빠른 C, J, A를 선택하여 "RCJA"가 됩니다.
