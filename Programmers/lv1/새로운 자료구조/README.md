# 문제 설명

정수를 저장할 수 있는 자료구조 S가 있습니다. 이 자료구조에는 command 명령이 있고, 정수 N에 대하여 다음과 같이 동작합니다.

**command N**: S에 N이 없으면 추가하고 있으면 삭제한다.

비어있는 S에 순서대로 command를 실행할 정수들이 들어있는 `queries`가 매개변수로 주어질 때,  
전부 처리한 뒤 자료구조 S에 있는 정수의 개수를 return 하는 `solution` 함수를 작성해 주세요.

### 제한사항
- 1 ≤ queries의 길이 ≤ 100,000
- 1 ≤ queries의 원소 ≤ 1,000,000,000

### 입출력 예
| queries          | result |
|------------------|--------|
| [0, 5, 17, 5, 1] | 3      |
| [1, 2, 1, 2]     | 0      |

### 입출력 예 설명

**입출력 예 #1**

예제 1번에서 비어있는 S에 queries에 주어진 순서대로 command 명령어를 처리하면 다음과 같습니다.

- command 0: S에 0이 없으므로 0을 추가합니다. **현재 S에는 0이 있습니다.**
- command 5: S에 5가 없으므로 5를 추가합니다. **현재 S에는 0과 5가 있습니다.**
- command 17: S에 17이 없으므로 17을 추가합니다. **현재 S에는 0, 5, 17이 있습니다.**
- command 5: S에 5가 있으므로 5를 삭제합니다. **현재 S에는 0, 17이 있습니다.**
- command 1: S에 1이 없으므로 1을 추가합니다. **현재 S에는 0, 17, 1이 있습니다.**

queries 순서대로 command 명령을 수행한 후 S에 0, 17, 1 **세 개의 정수**가 저장되어 있으므로 **3**을 return 합니다.

**입출력 예 #2**

예제 2번에서 비어있는 S에 queries에 주어진 순서대로 command 명령어를 처리하면 다음과 같습니다.

- command 1: S에 1이 없으므로 1을 추가합니다. **현재 S에는 1이 있습니다.**
- command 2: S에 2가 없으므로 2를 추가합니다. **현재 S에는 1과 2가 있습니다.**
- command 1: S에 1이 있으므로 1을 삭제합니다. **현재 S에는 2가 있습니다.**
- command 2: S에 2가 있으므로 2를 삭제합니다. **현재 S는 비어있습니다.**

queries 순서대로 command 명령을 수행한 후 S가 비어있으므로 **0**을 return 합니다.
