# [Programmers] [PCCE 기출문제] 9번 / 지갑 접기

### 문제 설명
지갑의 가로, 세로 크기를 담은 정수 리스트 `wallet`과 지폐의 가로, 세로 크기를 담은 정수 리스트 `bill`이 주어집니다. 지폐를 지갑에 넣기 위해서 지폐를 접어야 할 수도 있습니다. 지폐를 접는 규칙은 다음과 같습니다.

1. 지폐를 접을 때는 항상 길이가 더 긴 쪽을 반으로 접습니다.
2. 접은 후 소수점 이하 숫자는 버립니다. (정수 나눗셈 `//` 과 동일)
3. 지폐를 가로로 넣거나 세로로 넣었을 때 지갑의 크기보다 작거나 같다면 그만 접습니다.

지폐를 지갑에 넣기 위해서 최소 몇 번 접어야 하는지 return 하도록 solution 함수를 완성해 주세요.

---

### 제한사항
* `wallet`의 길이 = 2
* `bill`의 길이 = 2
* 10 ≤ `wallet`의 원소 크기 ≤ 100
* 10 ≤ `bill`의 원소 크기 ≤ 2,000

---

### 입출력 예
| wallet | bill | result |
| :--- | :--- | :--- |
| [30, 15] | [26, 17] | 1 |
| [50, 50] | [100, 241] | 4 |

---

### 입출력 예 설명
**입출력 예 #1**
* 지갑은 [30, 15]이고 지폐는 [26, 17]입니다.
* 처음 상태에서 지폐의 작은 쪽(17)이 지갑의 작은 쪽(15)보다 크므로 지폐를 접습니다.
* 지폐의 긴 쪽인 26을 반으로 접으면 [13, 17]이 됩니다.
* 이제 지폐의 작은 쪽(13)이 지갑의 작은 쪽(15)보다 작거나 같고, 긴 쪽(17)이 지갑의 긴 쪽(30)보다 작거나 같으므로 지갑에 들어갑니다.
* 따라서 1을 return 합니다.

**입출력 예 #2**
* 지갑은 [50, 50]이고 지폐는 [100, 241]입니다.
* 지폐를 접는 과정은 다음과 같습니다.
  1. [100, 241] -> [100, 120] (1번 접음)
  2. [100, 120] -> [100, 60] (2번 접음)
  3. [100, 60] -> [50, 60] (3번 접음)
  4. [50, 60] -> [50, 30] (4번 접음)
* 4번 접은 후 지폐의 크기가 [50, 30]이 되면 지갑 [50, 50]에 들어갈 수 있습니다.
* 따라서 4를 return 합니다.
